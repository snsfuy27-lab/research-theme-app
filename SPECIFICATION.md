# 10万語規模 学術用語辞書統合仕様書

## 1. システム概要とゴール

本システムは、高校生の課題研究テーマ作成を支援するWebアプリである。研究対象、測定語、操作条件、比較対象、分類基準などを入力すると、研究タイトルの文法構造、測定可能性、研究対象の具体性を即時評価する。

今回の拡張では、約12万語の学術用語辞書を統合し、辞書登録語を「十分に具体化された用語」として扱う。これにより、従来は総称語として誤検出されやすかった学術語、教科書語、測定語、具体的な生物名・物質名・統計用語・情報用語などを、研究対象または測定対象として正当に評価できるようにする。

最終目標は、10万語規模の辞書をWeb上で快適に検索・確認しながら、UI、検索性、評価ロジック、辞書語彙を継続的にチューニングできるプロトタイプを構築することである。

## 2. 画面遷移・UI/UX構成案

### 2.1 メイン入力画面

ユーザーは最初に研究型を選択する。

- 因果型
- 比較型
- 分類型
- 開発型

研究型に応じて入力欄を切り替える。各欄は、評価ルーブリックの観点と対応している。

### 2.2 研究タイトル生成画面

入力欄に基づいて、研究タイトル（仮）をリアルタイム生成する。タイトルはルーブリック評価より上に表示する。

### 2.3 ルーブリック評価画面

C1〜C5程度の観点で、研究対象の具体性、操作・比較・分類軸、測定語、研究構造を即時評価する。

### 2.4 測定語サジェスト画面

「成長」「効果」「大きい」「きれい」などの抽象語が入力された場合、測定可能な語に置換する候補を表示する。

### 2.5 学術用語辞書検索パネル

今回追加した画面である。約12万語の辞書をブラウザ上で検索できる。

主な要素は以下のとおりである。

- キーワード検索
- カテゴリ絞り込み
- 完全一致検索
- 表示件数切り替え
- 登録語の具体語判定表示
- カテゴリ、下位分類、用語の一覧表示

## 3. 機能要件・仕様詳細

### 3.1 10万語規模の高速検索

辞書CSVは `public/academic_terms_dictionary_100000.csv` に配置する。初回表示時にクライアント側でCSVを読み込み、検索パネル内で絞り込み表示する。

検索結果はすべてを一度に描画せず、表示件数を30件、80件、150件、300件から選択する。これにより、10万語規模でもDOM描画負荷を抑制する。

### 3.2 辞書登録語の具体語扱い

`lib/academic-term-data.ts` に辞書語の完全一致Setを生成している。入力語が辞書登録語に完全一致する場合、その語は具体性が担保された例外語として扱う。

この仕様により、例えば以下の語は総称語として過剰に警告されにくくなる。

- 酸素
- 力の大きさ
- 線形回帰
- ミドリゾウリムシ
- 酸素濃度
- カイワレダイコンの発芽率

### 3.3 インクリメンタル検索

検索欄の入力に応じて即時検索する。完全一致モードでは、入力語と辞書登録語が完全一致するものだけを表示する。

### 3.4 レスポンシブ対応

スマートフォンでの利用を前提とし、最大幅を抑えた1カラム構成を基本とする。辞書パネルは縦スクロール可能な表として表示する。

### 3.5 静的公開対応

Next.jsの `output: 'export'` を利用する。Vercelだけでなく、GitHub Pagesでも公開できる構成とする。

## 4. データ構造と技術スタックの提案

### 4.1 現在のプロトタイプ構成

- フロントエンド: Next.js + React + TypeScript
- UI: Tailwind CSS
- 状態管理: React hooks
- データ保存: localStorage
- 辞書データ: CSV + TypeScript Set
- 公開: VercelまたはGitHub Pages

### 4.2 辞書CSV構造

```csv
category,subcategory,term,specificity_status,specificity_note
```

各列の意味は以下のとおりである。

- category: 大分類
- subcategory: 下位分類
- term: 用語
- specificity_status: 具体性判定上の扱い
- specificity_note: 評価ロジック上の説明

### 4.3 将来の推奨構成

10万語規模を継続的に編集する場合、最終的には以下の構成が望ましい。

- フロントエンド: Next.js
- 辞書検索: Fuse.js、MiniSearch、FlexSearchのいずれか
- データ保持: Supabase、Turso、SQLite、またはVercel Postgres
- 管理画面: 辞書語の追加、削除、カテゴリ変更、具体性タグ変更
- データ出力: CSV、JSON、TS辞書ファイルの自動生成

初期検証段階では、現在のCSV静的配信方式が最も簡単である。GitにCSVを置くだけで公開でき、VercelでもGitHub Pagesでも扱いやすい。

## 5. 確認事項

1. 辞書登録語はすべて「具体語」として扱う方針でよいか。ただし「植物」「社会」「文化」のような総称語を辞書に含める場合、例外的に総称語扱いへ戻す必要がある。
2. 10万語辞書は、研究テーマ評価用の具体語辞書として使うのか、単なる検索辞書としても使うのか。
3. 次段階で、辞書語ごとに「研究対象語」「測定語」「操作条件語」「分類基準語」「総称語」のタグを付与するか。

この仕様で良ければ「開発開始」と指示してください。次回、コードとGit公開（Vercel等のデプロイ対応）手順を一括出力します。
